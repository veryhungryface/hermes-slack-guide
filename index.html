Hermes Agent Slack 연동 가이드

이 가이드는 Hermes Agent에서 Slack 채널을 읽고 메시지를 보내기 위해 필요한 설정 및 사용 방법을 설명합니다.

1. Slack API 토큰 발급

Slack API를 사용하려면 사용자 토큰(User Token, `xoxp-`로 시작)이 필요합니다. 이 토큰은 워크스페이스의 모든 채널에 대한 읽기 및 쓰기 권한을 가질 수 있으므로 보안에 각별히 유의해야 합니다.

토큰 발급 방법 (간략 가이드):

  • Slack API 웹사이트 ([https://api.slack.com/apps](https://api.slack.com/apps))에 접속하여 새 앱을 생성합니다.

    • 2. `From scratch`를 선택하고 앱 이름과 워크스페이스를 지정합니다.

    3. `Features` > `OAuth & Permissions` 메뉴로 이동합니다.

    4. `Scopes` 섹션에서 다음 권한을 추가합니다.

  • `channels:read` (채널 목록 조회)
  • `channels:history` (채널 메시지 읽기)
  • `chat:write` (메시지 보내기)
  • `groups:read` (비공개 채널 목록 조회)
  • `mpim:read` (그룹 DM 목록 조회)
  • `im:read` (개인 DM 목록 조회)

    • 5. `OAuth & Permissions` 상단의 `Install to Workspace` 버튼을 클릭하여 앱을 설치합니다.

    6. 설치 후 `OAuth Tokens for Your Workspace` 섹션에서 `xoxp-`로 시작하는 User OAuth Token을 복사합니다.

    🚨 보안 경고: 이 토큰은 외부에 노출되지 않도록 철저히 관리해야 합니다. Hermes Agent에서는 메모리에 저장하는 것을 권장하지 않지만, 사용자 지시에 따라 임시로 저장할 수 있습니다. 실제 프로덕션 환경에서는 환경 변수 등을 활용하는 것이 안전합니다.

    2.4. Slack 채널에 메시지 보내기

    특정 채널에 메시지를 보내는 예시입니다. `mrkdwn` 문법을 사용하여 서식을 지정할 수 있습니다.

    

import shlex
from hermestools import terminal
import json

slacktoken = "xoxp-1...0676" # 실제 토큰으로 교체
channelid = "YOURCHANNELID" # 메시지를 보낼 채널 ID로 교체
messagetext = "안녕하세요! Hermes Agent에서 보낸 테스트 메시지입니다. 강조 표시도 가능합니다."

postmessagepayload = {
    "channel": channelid,
    "text": messagetext,
    "mrkdwn": True
}

postmessagecommandparts = [
    "curl".
    "-s".
    "-X", "POST".
    "-H", shlex.quote(f"Authorization: Bearer     "-H", "Content-type: application/json; charset=utf-8".
    "-d", shlex.quote(json.dumps(postmessagepayload, ensureascii=False)),
    shlex.quote("https://slack.com/api/chat.postMessage")
]
postmessagecommand = " ".join(postmessagecommandparts)

result = terminal(command=postmessagecommand, timeout=60)

print(result["output"])

    3. 요약 및 주의사항

  • Slack API 토큰(`xoxp-`)은 강력한 권한을 가지므로 절대 외부에 노출되어서는 안 됩니다.
  • Hermes Agent의 `terminal` 도구를 통해 `curl` 명령어를 사용하여 Slack API와 직접 상호작용할 수 있습니다.
  • `shlex.quote`를 사용하여 `curl` 명령어의 인수를 안전하게 처리하는 것이 중요합니다.
  • Slack API 문서를 참조하여 더 다양한 기능(`conversations.replies`, `users.info` 등)을 활용할 수 있습니다.
