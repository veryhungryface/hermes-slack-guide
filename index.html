Hermes Agent Slack 연동 가이드

App Configuration Token과 Slack CLI로 앱 생성, 권한 스코프, 설치 URL 발급, 채널 조회, 메시지 읽기와 보내기까지 최대한 터미널에서 처리하도록 정리했습니다.

연동 흐름

CLI + Manifest

  2. 02 Manifest 작성봇 토큰과 유저 토큰 권한을 JSON으로 고정한다.

Step 1

CLI로 Slack 앱 만들기

웹에서 반복 클릭하는 대신, 한 번만 Slack Apps의 Your App Configuration Tokens에서 config token을 발급하고 나머지는 Slack CLI로 처리합니다. App Configuration Token은 앱별 토큰이 아니라 워크스페이스와 사용자에 묶인 토큰이며, App Manifest API 호출에 사용합니다.

  • 웹에서 하는 일은 config token과 refresh token을 발급하는 것까지로 제한합니다.
  • slack api apps.manifest.validate로 manifest를 먼저 검증합니다.
  • slack api apps.manifest.create로 앱을 생성하고, 응답의 oauth_authorize_url로 설치합니다.
  • 설치 후 발급된 xoxb- 봇 토큰과 xoxp- 유저 토큰을 환경 변수로 분리해 사용합니다.

설치 승인은 아직 필요합니다. 앱 생성과 권한 구성은 CLI로 끝내고, 마지막 workspace authorize 화면만 열어 승인합니다. Slack manifest의 bot_user.display_name은 소문자, 숫자, 하이픈 등으로 제한되어 예시는 ism-helper로 두었습니다.

Step 2

필요 스코프

이 manifest는 봇 토큰과 유저 토큰을 둘 다 쓰는 운영을 기준으로 합니다. 봇 토큰은 앱이 참여한 대화와 앱 동작에, 유저 토큰은 권한을 승인한 사용자가 볼 수 있는 채널/검색/DM 조회에 씁니다.

토큰 필수 스코프
봇 토큰 app_mentions:read, chat:write, files:read, files:write, im:history, im:read, im:write, reactions:write, users:read
유저 토큰 channels:history, channels:read, chat:write, groups:history, groups:read, im:history, im:read, mpim:history, mpim:read, search:read, users:read
Config 토큰 Slack 앱을 만들고 수정하는 App Manifest API 전용 토큰입니다. 봇/유저 Web API 호출에는 사용하지 않습니다.

Manifest 구조는 Slack의 App manifest reference, 앱 생성 API는 apps.manifest.create, CLI 인자 형식은 slack api 기준입니다.

Step 3

토큰 보관 원칙

토큰은 문서, 코드, 채팅 기록에 남기지 마세요. Hermes에게 Slack 작업을 맡길 때도 실제 토큰 값은 환경 변수나 시크릿 저장소를 통해 주입하는 방식이 안전합니다.

권장 방식

  • SLACK_APP_CONFIG_TOKEN은 앱 생성/수정에만 사용
  • SLACK_BOT_TOKEN과 SLACK_USER_TOKEN 분리
  • config token은 12시간 만료 전에 refresh token으로 회전

피할 방식

  • manifest, README, HTML에 실제 토큰 기록
  • Agent 메모리에 장기 저장
  • config token을 Slack Web API 호출용으로 재사용
Step 4

Hermes에서 실행할 명령

Hermes Agent의 터미널 실행 도구에서 아래 명령을 그대로 사용할 수 있습니다. 조회는 유저 토큰, 앱 이름으로 쓰기/파일/반응은 봇 토큰을 기본으로 나눕니다. 응답의 ok 값이 true인지 먼저 확인하세요.

Step 5

Agent에게 맡길 때

Hermes에게는 작업 의도와 채널 ID, 그리고 어떤 토큰을 써야 하는지만 전달합니다. 실제 토큰 값은 이미 설정된 환경 변수로 읽게 하는 흐름이 가장 깔끔합니다.

출력 규칙을 같이 주세요. 예를 들어 "원문 전체를 붙이지 말고 결정 사항만 요약"처럼 범위를 제한하면 민감한 대화가 불필요하게 확산되는 일을 줄일 수 있습니다.

Codex skill로 앱 생성

이 repo에는 로컬 GUI 스킬도 포함되어 있습니다. App Configuration Token, 앱 이름, 권한 체크박스로 manifest를 만들고, 설치 후 개인 Slack 작업용 Codex skill 생성 지침까지 한 번에 만들 수 있습니다.

개인 머신 전용으로 쓰세요. GUI가 생성하는 개인 Slack 작업 스킬에는 입력한 앱 ID와 유저 토큰을 넣을 수 있습니다. 생성된 개인 스킬 폴더는 GitHub나 공유 드라이브에 올리지 마세요.

Reference

자주 막히는 지점

  • not_in_channel: 앱을 해당 채널에 초대하세요. 비공개 채널은 초대가 필수입니다.
  • missing_scope: manifest의 oauth_config.scopes를 수정한 뒤 apps.manifest.update 또는 재생성 후 다시 설치하세요.
  • invalid_auth: 토큰 값, 워크스페이스, 토큰 회전 여부를 확인하세요.
  • invalid_manifest: apps.manifest.validate 응답의 errors[].pointer가 가리키는 필드를 고치세요.
  • 목록에 채널이 없음: types 값과 앱이 접근 가능한 대화 범위를 확인하세요.
  • 긴 스레드가 누락됨: conversations.replies로 스레드 답글을 별도로 조회하세요.
